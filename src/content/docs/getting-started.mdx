Maester is a Node CLI. You install it once, then use it to manage two
kinds of configuration files: a `citadel.yaml` that lists the sources you
pull from, and (optionally) a `maester.yaml` on each source that declares
what it publishes.

## Install

```bash
npm install -g @baller.software/maester
```

If you'd rather not install globally, every command below also works
through `npx`:

```bash
npx @baller.software/maester init
```

## Register your first citadel

Run the interactive walk-through from the root of the repository you
want to use as an aggregation point:

```bash
maester init
```

It will:

1. Prompt for one or more source repositories to register.
2. Per source, offer an optional `includes` list — leave it blank to
   defer to the remote's own `maester.yaml`, or declare paths/globs
   directly when the remote has no manifest (a third-party repo, say).
3. Validate any `auth.type` env-var names so secrets never land in the
   `citadel.yaml`.
4. Add an idempotent `maester:sync` script to `package.json` if one
   exists.

When it's done, you'll have a `citadel.yaml` at the repo root. Commit it.

## Sync the citadel

To pull every source in your `citadel.yaml` into the local tree:

```bash
maester sync
```

Or to pull a single named source:

```bash
maester sync source-context
```

Sync uses partial clones and sparse checkouts — it only fetches the
paths the source actually publishes.
