Maester is configured through two YAML files: `citadel.yaml` lives at
the root of an aggregating repository, and `maester.yaml` lives at the
root of each source repository.

## citadel.yaml

The aggregator's configuration. Lists the sources to pull from.

```yaml
schemaVersion: 1
baseDir: context           # optional; defaults to 'citadel'
sources:
  # Manifest-driven: the remote publishes its own maester.yaml.
  - name: source-context
    url: https://github.com/your-org/source-repo
    ref: main
    destination: docs/external/source-context   # optional override
    auth:
      type: token
      envVar: GH_TOKEN
  # Includes-driven: the citadel declares what to pull.
  - name: third-party
    url: https://github.com/another/repo
    ref: main
    includes:
      - 'README.md'
      - 'docs/**/*.md'
```

### Top-level fields

| Field           | Type     | Required | Description                                                                 |
| --------------- | -------- | -------- | --------------------------------------------------------------------------- |
| `schemaVersion` | integer  | yes      | Schema version. Current version is `1`.                                     |
| `baseDir`       | string   | no       | Parent folder for sources whose `destination` is unset. Default: `citadel`.  |
| `sources`       | list     | yes      | The remote git repositories this citadel pulls from. At least one entry.    |

### Per-source fields

Every entry in `sources[]` accepts:

- `name` — unique slug; becomes the destination folder name.
- `url` — git URL (`https://`, `ssh://`, `file://`, or `git@host:path`).
- `ref` — optional branch, tag, or commit SHA. Defaults to the remote's
  default branch.
- `includes` — optional list of glob patterns. **When set, the citadel
  owns the filter set and the remote `maester.yaml` is not consulted.
  When unset, sync looks for a `maester.yaml` on the remote and uses its
  paths.** An empty `includes` array is rejected.
- `destination` — optional override for the output path. Wins over
  `baseDir`.
- `auth` — optional credentials object. `auth.envVar` names the env var
  that holds the token; the value itself never lands in YAML.
- `description`, `tags` — optional metadata surfaced in `--verbose`
  output.

The two source modes — manifest-driven (no `includes`) and includes-driven
(`includes` set) — are processed by the same sync command. Use
includes-driven sources for public third-party repos or repos you have
read access to but do not own.

## maester.yaml

The publish manifest a source repository ships. Lists the documentation
the source is willing to share.

```yaml
schemaVersion: 1
documents:
  - path: README.md
    category: readme
    description: Project overview.
  - path: docs/architecture.md
    category: reference
  - path: docs/recipes/**/*.md
    tags: [recipes, advanced]
```

### Fields

- `schemaVersion` — current version is `1`.
- `documents[]` — the publish list. Each entry declares a `path` (file
  or glob, repo-relative) and optional `description`, `category`, and
  `tags`.

The publish manifest doesn't enforce a category taxonomy. Citadels
choose whether to surface those hints in their consumer.

## Generating either file

You don't have to write these by hand:

```bash
maester init      # interactive citadel.yaml walk-through
maester publish   # interactive maester.yaml walk-through
```
